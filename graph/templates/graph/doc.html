Benchmark concentration modeling web application

A. Purpose of this application

The benchmark concentration modeling tool determines a BMC (benchmark concentration) from in-vitro cytotoxicity data, and it calculates the lower limit of the 95% confidence interval (BMCL15) for the BMC15 (benchmark concentrations associated with a benchmark response of 15% (= 85% viability)). As further output, the renormalized data points can be downloaded and the parameters for the mathematical curve describing the cytotoxicity can be obtained.

⇒The BMCL15 can be used to define non-cytotoxic concentrations, i.e. all concentrations < BMCL15.

B. Short tutorial of the 4 steps required to operate the BMC modeling tool:

  1. Upload experimental concentration-response data
  2. Click on ‘Update’ and choose benchmark response (BMR)
  3. Check graph for plausibility and chose display options for data
  4. Download graph and BMCL

C. Explanation of buttons and menu parts:

Login

To Login please provide an email address and a self-chosen password. Every time you login with your user data the datasets from your previous session are loaded.

Logout

By clicking the logout button all the processed data from the current session are saved and can be readily accessed in the next login.

‘Input data’ menu:

Data can be entered manually, or a file can be uploaded. The data format is simple and uniform: each line defines one data point. Each line has three columns.

The three columns are used as follows:

First column: concentration (C)

Second column: response (R) (= viability)

Third column: Number of the experiment (#)

C: concentrations (in nM) that were used for the treatment of the cells (e.g. compound concentration of arsenic was 0.001 nM). R: has to be given as percentage of surviving cells. This means that the user has to subtract the background from the raw data and normalize the data with untreated controls prior to benchmark concentration modeling. #: input is an integer number, to define from which experiment the concentration-response data are derived. This allows analysis of several samples (of the same treatment) in the same file. If only a single experiment is analyzed this number is 1(If the same number is used more than once for a given concentration, this is interpreted as technical replicate within one experiment).

Example for data input lines:

doseresponseexperiment
0.0000196.321(experiment 1)
0.0001123.511
0.00192.631(second technical replicate of experiment 1)
0.00192.631(experiment 2)
0.0197.361

New custom sample:

By pushing the new custom sample button you can specify a name for your sample and manually insert the data in the grid.

Browse:

Input data have to be provided in a tab delimited text file format. By clicking on the ‘browse’ button (might be displayed in your language depending on your browser settings), you can search the file system of your computer and upload the file. Data from Excel can be saved as tab delimited text files. After upload, the data show up in the grid on the left side of the screen. To start the modeling procedure you must click on Update as described below.

Update:

Clicking on the ‘Update’ button runs the algorithm in the background. The graph with the fitted model then pops up in the middle of the screen. Data points, for example outliers, can be edited at this stage, or additional data can be added (by clicking the green plus and removed by clicking on the red cross)

• The data in the grid are now re-normalized. This means there are different response data now in the grid. This is the effect of renormalization across the entire data set (The final renormalized data can be downloaded as .txt files by clicking on the text icon in the edit sample menu as described in detail under “edit samples".

• After each modification ‘Update’ has to be clicked again to re-run the algorithm on the edited dataset.

Dynamic Graph:

After pushing the ‘Update’ button, a graph pops up that shows the fitted model and the normalized data from each experiment in different colors. Items can be removed from the graph by clicking on the line and point symbols in the legend under the graph. In this manner each single experiment can be visualized alone or together with others. Moreover, an average curve (obtained by fitting the model to the pooled data) for all experiments can be displayed as black dashed line. The average curve is used for calculation of the BMC and BMCL, and it is obtained by fitting the optimal model again through the normalized datasets pooled from all experiments.

Output information

Above the graph, the benchmark concentration (BMC) and the benchmark concentration lower limit (BMCL) are depicted for a benchmark response (BMR) of 50%, 15% and 10% respectively.

Under the graph, the chosen model and its parameters are displayed (self-starter functions as described in detail in the R package drc, Ritz, C. & Streibig, J. C. (2005)):

W1.4

f(x) = c + (d-c) exp(-exp(b(log(x) -log(e))))

W2.4

f(x) = c + (d-c)(1 - exp(-exp(b(log(x) -log(e)))))

W1.2

f(x) = exp(-exp(b(log(x) -e)))

W1.3

f(x) = 0 + (d-0) exp(-exp(b(log(x)-e)))

LL.5

f(x) = c +d - c(1 + exp(b(log(x) -log(e))))f

LL.4

f(x) = c +d-c(1 + exp(b(log(x) -e)))f

LL.3

f(x) = c +1-c (1 + exp(b(log(x) -e))

LL.2

f(x) =1 + exp(b(log(x) -log(e)))

This area is also used as alert for errors that occurred during modeling and to indicate anchor points introduced by the software.

The algorithm sets an anchor point at y=0, and at x >> BMC if there are not sufficient data in the toxic range for meaningful curve modeling. As demonstrated empirically, this procedure allows still for reasonable estimates of BMCL10 or BMCL15, but it should not be used for estimates of BMC50/BMCL50. The algorithm calculates the EC50*80 and the highest treatment dose*80. The smaller one of these two values is used as anchor point and the x-coordinate is also shown in the area below the graph. It can be distinguished from measured data points because it is emphasized in larger symbol size and black.

The graph can be downloaded in svg, png, jpg and pdf by expanding the menu and clicking on the desired format on the upper right corner of the graph.

Samples to display:

On the right side of the graph you have the possibility to choose between your samples by moving your cursor and clicking on checkbox next to the desired sample. This sample is visualized in the graph field now and the corresponding data are shown in the grid.

Edit sample:

If you want to modify a previously uploaded sample you have to select this sample in the edit sample menu. Right of the ‘edit sample’ button, there is a text icon. Clicking it allows download of the normalized data. Next to that text icon, there is a graph icon. Clicking it allows download of a .zip file (you need to download winzip, 7zip or any other program able to open this file format to decompress). This folder contains all the single fitting steps from the algorithm as described in the paper (put link to my paper here).

Clear all:

Clicking on the “Clear All” button removes all previously uploaded datasets. You are asked if you are sure that you want to delete everything when you push this button.
